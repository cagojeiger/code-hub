## 설계 문서 정합성 검토

구현한 코드가 다음 문서들과 일치하는지 확인하세요:

### 1. spec.md 검토
- API 엔드포인트 시그니처 일치
- 에러 코드 (INVALID_STATE, WORKSPACE_NOT_FOUND 등)
- 상태 전이 규칙 준수
- Config 구조 일치

### 2. architecture.md 검토
- 컴포넌트 책임 분리
- 데이터 흐름 패턴
- 상태 × 액션 매트릭스 준수

### 3. ADR 검토
docs/adr/*.md 결정사항 확인:
- 기술 스택 선택 (ADR-001)
- 프로젝트 구조 (ADR-002)
- 기타 아키텍처 결정

### 4. 체크리스트
- [ ] 새로운 API가 spec.md에 정의되어 있는가?
- [ ] 에러 응답 형식이 spec과 일치하는가?
- [ ] 컴포넌트 경계를 넘는 직접 호출이 없는가?
- [ ] ADR에서 금지한 패턴을 사용하지 않았는가?

---

**위배 발견 시**:
- 코드 수정이 맞으면 → 코드 수정
- 스펙 변경이 필요하면 → spec 수정 PR 먼저
- 아키텍처 결정 필요 → ADR 작성 (Human 승인)
